# 프론트엔드 과제: 턴제 배틀 게임

## 개요

턴제 배틀 게임을 구현합니다. 플레이어는 캐릭터를 세팅한 뒤, 적 캐릭터와 1:1 턴제 전투를 벌입니다.

**기간**: 7일

**기술 스택**: 프레임워크, 상태 관리, 스타일링 등 자유롭게 선택

아래는 게임 화면의 예시입니다. 반드시 동일하게 구현할 필요는 없으며, 참고용으로 제공됩니다.

## 게임 흐름

게임은 **세팅 → 전투 → 결과**로 구성됩니다.

---

## 1. 세팅

세팅은 3단계 스텝 폼으로 진행됩니다. 각 스텝 간 이동(이전/다음) 시 입력한 값이 보존되어야 합니다.

### Step 1: 이름 설정 및 스탯 배분

**이름**
- 1~10자 제한

**스탯 배분**
- 총 **200포인트**를 5개 스탯에 배분합니다.
- 각 스탯은 아래 최소/최대 범위 내에서 배분해야 합니다.
- 잔여 포인트가 실시간으로 표시되어야 합니다.
- 배분 완료(잔여 0) 시에만 다음 스텝으로 진행할 수 있습니다.

| 스탯 | 최소 | 최대 |
|---|---|---|
| HP | 20 | 100 |
| MP | 20 | 100 |
| ATK | 5 | 30 |
| DEF | 5 | 30 |
| SPD | 5 | 30 |

> 예시: HP 100, MP 50, ATK 20, DEF 15, SPD 15 = 200포인트

### Step 2: 스킬 설정

**기본 스킬 (삭제 불가)**

아래 2개의 기본 스킬은 항상 보유하며, 삭제할 수 없습니다.

| 스킬 | 타입 | MP 소모 | 효과 |
|---|---|---|---|
| 공격 | attack | 0 | ATK × 1.0 배율 데미지 |
| 방어 | defend | 0 | 이번 턴 받는 피해 50% 감소 |

**커스텀 스킬 생성**

기본 스킬 외에 **최대 2개**의 커스텀 스킬을 생성할 수 있습니다 (기본 2개 + 커스텀 2개 = **최대 4개 스킬**).

커스텀 스킬은 아래 4개 타입 중 하나를 선택하여 생성합니다. 타입에 따라 입력해야 하는 필드가 달라집니다.

**공통 필드**
- 이름: 1~8자
- MP 소모: 1~30

**타입별 필드**

| 타입 | 추가 필드 | 범위 |
|---|---|---|
| attack | 배율 | 1.0 ~ 3.0 |
| heal | 회복량 | 10 ~ 50 |
| buff | 대상 스탯 (ATK/DEF), 수치, 지속 턴 | 수치: 1 ~ 10, 턴: 1 ~ 5 |
| debuff | 대상 스탯 (ATK/DEF), 수치, 지속 턴 | 수치: 1 ~ 10, 턴: 1 ~ 5 |

> 예시: "분노" (buff, ATK +5, 3턴, MP 8) → 3턴 동안 자신의 ATK에 +5 가산. "위협" (debuff, ATK -5, 3턴, MP 10) → 3턴 동안 상대의 ATK에서 5 차감.

생성한 커스텀 스킬은 자유롭게 삭제할 수 있습니다.

### Step 3: 난이도 설정

3가지 난이도 중 하나를 선택합니다.

| 난이도 | 설명 |
|---|---|
| 쉬움 | 약한 적 |
| 보통 | 보통 적 |
| 어려움 | 강한 적 |

난이도 선택 후 전투를 시작합니다.

## 2. 전투

### 턴 구조

- 매 라운드 **SPD가 높은 쪽**이 선공합니다 (동일 시 랜덤).
- 한 라운드에서 양쪽 캐릭터가 각각 1회씩 행동합니다.
- 행동 중 상대 HP가 0 이하가 되면 전투가 즉시 종료됩니다.
- 세부 턴 진행 순서는 자유롭게 설계하세요.

### 행동

플레이어는 매 턴 다음 중 하나를 선택합니다:

| 행동 | 효과 |
|---|---|
| 공격 | 기본 공격 (배율 1.0, MP 소모 없음) |
| 방어 | 이번 턴 피해 50% 감소 (MP 소모 없음) |
| 스킬 사용 | 보유 스킬 중 하나 사용 (MP 소모) |

### 데미지 공식

```
damage = (공격자 ATK × 배율 - 상대 DEF × 0.5) × (상대가 방어 중이면 0.5, 아니면 1.0)
```

- ATK/DEF는 버프/디버프가 반영된 **유효 스탯** 기준
- **배율**: 기본 공격 = 1.0, 스킬은 스킬별 배율
- 최소 데미지 1 보장

### 스킬 효과

| 타입 | 효과 |
|---|---|
| attack | 데미지 공식 참고 |
| heal | HP 회복 (최대 HP 초과 불가) |
| buff | 자신의 ATK 또는 DEF 증가 (지속 턴만큼 유지) |
| debuff | 상대의 ATK 또는 DEF 감소 (지속 턴만큼 유지) |

### 버프 / 디버프

- 버프와 디버프는 일정 턴이 지나면 만료됩니다.
- 유효 ATK/DEF는 기본값에 버프/디버프를 반영하여 계산합니다.

### 전투 종료 조건

| 조건 | 결과 |
|---|---|
| 플레이어 HP ≤ 0 | 패배 |
| 적 HP ≤ 0 | 승리 |
| 20턴 초과 | 무승부 |

## 3. 적 캐릭터

### 난이도별 스탯 (고정값)

| 스탯 | 쉬움 | 보통 | 어려움 |
|---|---|---|---|
| HP | 80 | 110 | 140 |
| MP | 30 | 50 | 70 |
| ATK | 10 | 15 | 20 |
| DEF | 8 | 12 | 16 |
| SPD | 7 | 10 | 14 |

### 난이도별 스킬 (고정)

| 난이도 | 보유 스킬 |
|---|---|
| 쉬움 | 공격, 방어, 강타 (attack, 배율 1.3, MP 8) |
| 보통 | 공격, 방어, 강타 (attack, 배율 1.5, MP 10), 회복 (heal, 회복량 20, MP 10) |
| 어려움 | 공격, 방어, 강타 (attack, 배율 1.7, MP 12), 회복 (heal, 회복량 30, MP 12), 약화 (debuff, DEF -5, 3턴, MP 10) |

### 행동 패턴

난이도별로 적의 행동 성격이 다릅니다. 구체적인 판단 로직은 자유롭게 구현하되, 아래 특성이 반영되어야 합니다.

- **쉬움**: 단순하게 행동합니다. 스킬을 적극적으로 활용하지 않습니다.
- **보통**: 상황에 따라 공격과 회복을 적절히 사용합니다. HP가 낮을 때 방어적으로 행동합니다.
- **어려움**: 전략적으로 행동합니다. 디버프를 활용하고, 위기 시 방어적으로 플레이합니다.

## 4. 전투 UI 요구사항

### 필수

- **캐릭터 정보**: 양쪽 캐릭터의 이름, HP/MP, 현재 적용 중인 버프/디버프 표시
- **행동 선택**: 공격, 방어, 스킬 버튼 (MP 부족 시 비활성화)
- **전투 로그**: 매 턴의 행동과 결과를 텍스트로 표시
- **결과**: 승패(또는 무승부) 여부, 총 턴 수 표시, 다시 시작하기

## 5. data-testid 명세

자동화된 기능 검증을 위해, 아래 `data-testid` 속성을 해당 요소에 부여해 주세요.

```html
<!-- 예시 -->
<input data-testid="name-input" />
<button data-testid="next-button">다음</button>
<span data-testid="remaining-points">120</span>
```

### 세팅 화면

| 요소 | data-testid |
|---|---|
| 캐릭터 이름 인풋 | `name-input` |
| 스탯 숫자 인풋 (각각) | `stat-hp`, `stat-mp`, `stat-atk`, `stat-def`, `stat-spd` |
| 잔여 포인트 숫자 | `remaining-points` |
| 다음 스텝 버튼 | `next-button` |
| 이전 스텝 버튼 | `prev-button` |
| 스킬 추가 버튼 | `add-skill-button` |
| 스킬 이름 인풋 | `skill-name-input` |
| 스킬 생성 확인 버튼 | `submit-skill-button` |
| 커스텀 스킬 삭제 버튼 | `remove-skill-button` |
| 난이도 선택 버튼 | `difficulty-easy`, `difficulty-normal`, `difficulty-hard` |
| 전투 시작 버튼 | `start-battle-button` |

### 전투 화면

| 요소 | data-testid |
|---|---|
| 라운드 표시 | `round-display` |
| 플레이어 패널 | `player-panel` |
| 적 패널 | `enemy-panel` |
| 플레이어 이름 | `player-name` |
| 적 이름 | `enemy-name` |
| 스킬 버튼 (순서대로) | `skill-button-0`, `skill-button-1`, ... |
| 전투 로그 영역 | `battle-log` |

### 결과 화면

| 요소 | data-testid |
|---|---|
| 결과 텍스트 (승리/패배/무승부) | `result-title` |
| 턴 수 표시 | `result-turns` |
| 다시 시작 버튼 | `restart-button` |
